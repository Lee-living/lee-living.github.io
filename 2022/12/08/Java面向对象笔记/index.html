构造方法

如果不主动写任何构造方法，那么默认会生成一个空参构造；
如果定义了构造方法，就不会生成默认的无参构造，如果写了有参但不手动写无参，就只能调用有参构造。
有参构造是在创建对象时就直接赋值，就不用一个一个调用set方法赋值。
一般类中 空参构造 有参构造都要写，应对不同使用场景。

标准JavaBean

我觉得不用过分笔墨，几条原则 :

类名 驼峰命名
所有成员变量都是用private修饰
提供无参构造方法，也提供带全参的构造方法
给每一个成员变量设定对应的get()/set()方法

idea一键标准JavaBean : FIle - Settings - Plugins - Marketplace - 搜索ptg - installed - 定义好成员变量后直接右键 Ptg to JavaBean.
