Java IO

IO是指Input/Output，即输入和输出，或读和写

Input是从外部读入数据到内存，例如，把文件从磁盘读取到内存，从网络读取数据到内存；读到内存并以如int[],byte[],String等格式让Java能处理。

Output是把数据从内存输出到外部，例如，把数据从内存写入到文件，把数据从内存输出到文件等。

InputStream / OutputStream

InputStream / OutputStream 的IO流以byte（字节）为最小单位，因此也称为字节流。

在Java中，InputStream代表输入字节流，OuputStream代表输出字节流，这是最基本的两种IO流。

Reader / Writer

如果我们需要读写的是字符，并且字符不全是单字节表示的ASCII字符，那么，按照char来读写显然更方便，这种流称为字符流。

Java提供了Reader和Writer表示字符流，字符流传输的最小数据单位是char。

差异

按照字节流的方式读取数据，无论任何类型都能读取， 每次只取一个字节/8个二进制位；例如目标的一个”a”，那么一次读取就可以取回，因为字母在内存中占一个字节/8位；如果目标是一个中文汉字，如”大”，那么就要分两次，因为中文通常是两字节/16位；其他类型如图片声音等文件类似。

而如果按照字符流的方式读取数据，只能读取文本文件，无论是字母”a”汉字”大”，都是一个字符，都可以一次取回，但其他类型文件就不适用了。

所以究竟使用字节流还是字符流，要取决于具体的使用场景。

Java.io 下的常见流

FileInputStream

文件字节输入流，万能的，任何类型的文件都可以采用这个流来读

FileInputStream就是以字节的方式，完成输入的操作，完成读的操作(磁盘 —> 内存)

如果用循环的方式来进行，就可以实现整个文件的读取

读到文件末尾时read()方法会返回-1，所以可以用以下方式实现整个文件的读取

但如果文件稍大，这样读取效率太低，我们可以通过字节组来读取数据

FIleOutputStream

文件字节输出流，负责写，从内存到磁盘

FIleReader

文件字符输入流，只能读取普通文本

使用方法和FileInputStream类似，不多赘述

FileWriterTest

文件字符输出流，只能输出普通文本

FileInputStream + FileOutputStream

FileInputStream + FileOutputStream可以一边读一边写实现文件的拷贝，使用字节流进行拷贝可以拷贝任何类型的文件

FIleReader + FileWriter

FIleReader + FileWriter只能拷贝普通文本文件，其他类似
